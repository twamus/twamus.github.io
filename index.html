qDIR64

The original qDIR was the name of an in-house file/data management tool written in qBASIC, used to process and manage mailing data lists. These utilities were written in Visual Basic and Python in around 2014.

As 16-bit applications were reaching their end of life, a lot of our old data management tools were going to be left in the dust. We needed a solution to solve the looming deprecation of older software solutions, retain compatibility with older files, and allow for the future growth and additions to tools without breaking the core system or drastic changes to the workflow.




Noteable Features
  • Fully controllable by keyboard navigation to speed up processing (selecting items with spacebar, tab to change to tools, using back arrow to swap between files and folders).
  • Customizable toolbar; could enable and disable tools with dragg and drop.
  • Purely modular; The core code was written in Visual Basic, but also supported tools written and scripts in Python and other languages.
Python Tool Descriptions
  • ADD (Python): Blindly combines multiple CSV files together. Useful for combining standardized lists.
  • DBF to CSV (Python): Converts DBIII file to CSV format
  • ODOA Test (Python): Strips line breaks out of the middle of CSV fields.
  • Satori Prep (Python): Prepares a CSV to be ran through an automated template in Satori Bulk Mailer. The script would run, prompt the user to run Satori address verification, Satori would export a file, and the script would attach the fields we did not want modified by Satori
  • Shuffle (Python): Used to prepare items for N-up printing. This would make it so that when printing and cutting a stack of postcards that they would be in the correct order.
  • Split (Python): This would split one CSV file into multiple CSV files by a field value.
  • Title Case (Python): Simply title cases all fields.
  • Uppercase (Python): Simply makes every field uppercase.
VB Tool Descriptions
  • DBQ (VB): Simple and quick CSV viewer and editor (useful for when Excel really liked dropped leading "0"s on zip codes)
  • LLoDD (VB): This would generate a suggested layout preview and breakdown/example of each fields to show the client and ask them what they would like on their mailing piece. This also generated .ADM files for use in inkjet address printers.
  • Merge (VB): Merges numerous CSVs of varying contents together into one clean master. The user decides which fields get merged together into the output file
  • Zip4 Finder (VB): Satori address checking would sometimes miss the mark on address checking. This tool would pull from the USPS website to verify the Zip+4, and then export it to an address record in Satori Bulk Mailer.

Project Management Tool

Due to the nature of proprietary information on this project, I am unable to share any screenshots or the name of it, thus I will be referring to it as Project Management Tool, or PMT for short. I do, however, have license to create a mock-up, from memory, in Microsoft Paint!




Initially, the workflow was based off of Excel documents that would get updated throughout the day/week, which was a logistical nightmare to ensure everyone was working off of the same document. This tool was created as a place to centrally host work orders relating to clients that would ship to multiple stores nation wide. Many of the orders would have similar base components, so there were templates and presets setup to help streamline the process. This was a fully functional project management tool, as well as full integration with packing, shipping, and tracking.

The end scope of this project was to house all of the data for each of the projects so that things such as packing and shipping could be automated. This platform was built with a MAPP stack (Mac Server, Apache, PostgreSQL, PHP). I have also listed the shipping tool as its own project in my projects list.

Project Management (Core) Features:
  • Export items to Excel spreadsheet
  • Billing amortization of company's fees based on percentage of billable items per store and project
  • Presets of common items and ability to copy items to other stores or tabs allow quickly adding items
  • Multiple budgeting sheets
Packing and Shipping Integration Features:
  • Packing list generation that generated QR code to be utilized as a unique package identifier.
  • Override in packing system if shipments needed to go to be shipped to a different location.
  • Tracking report generation to provide to project managers at the end of business day.
  • Package delivery status and location of items from freight and domestic shipping carriers updated automatically.

Shipping Tool

This was a shipping tool that was designed to complete the loop between the packing system and the project management tool, and was intended to be used as the primary shipping method. The sizing and spacing of fields may appear a bit off due to a mix of window preview, being loaded into a newer version of Visual Studio, and/or a newer version of .NET.




As I never officially launched this project into active production, I can fortunately share a bit more details than the project management tool. All options are fully functional unless being noted as being in development.

Shipping-related features:
  • Integration to ship domestically utilizing FedEx Web Services (project was certified)
  • Ability to reprint labels or packing lists.
  • Mettler-Toledo Scale integration (other scales may work, but would likely need additional configuration)
  • Third party billing, saturday delivery, and direct signature required options.
  • The ability to filter items listed for jobs with a large number of items by the type of item or the item's ID.
Company specific features:
  • Generation and printing of colored thermal company proprietary routing labels.
  • Generation and printing of thermal packing lists.
  • Would update and highlight items that were being actively changed/modified by project managers using SQL LISTEN/NOTIFY triggers.
  • Presets of common box types that could be saved.
  • (Mid-development) Batch shipping using weight estimation for items of similar contents.
  • Automatic (based on project manager's request), manual, and location overrides for package destinations.
